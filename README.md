# SEC AAER Rare-Event Audit-Risk Screening

This repository contains the reproducible code and manuscript-ready outputs for a temporal external-validation study of SEC Accounting and Auditing Enforcement Release (AAER) firm-year audit-risk screening.

The workflow trains baseline machine-learning models, selects an AI teacher using the validation period, trains an AI-guided single-index varying-coefficient (VC) student model, compares against an unguided VC ablation, evaluates once on the held-out temporal test period, and exports publication tables and high-resolution figures.

## Repository structure

```text
data/
outputs/
scripts/
tested_outputs/
FINAL_RESULTS_SUMMARY.md
README.md
requirements.txt
```

- `data/`: expected location of the SEC AAER firm-year modeling dataset.
- `scripts/`: Python scripts for running the full analysis pipeline.
- `outputs/`: tables, figures, logs, and manuscript-ready results generated by the pipeline.
- `tested_outputs/`: verified outputs from a completed run.
- `FINAL_RESULTS_SUMMARY.md`: concise summary of the final results.
- `requirements.txt`: Python package requirements.

## Data

The main modeling file should be placed at:

```text
data/sec_aaer_firmyear_modeling_dataset_2009_2026.csv
```

The current package is configured for the verified SEC AAER firm-year dataset used in the manuscript run. To rerun the analysis with a newly rebuilt dataset, replace the CSV in `data/` before running the pipeline.

## Temporal validation design

The study uses a strict chronological split:

| Period | Years |
|---|---:|
| Training | 2009-2018 |
| Validation | 2019-2021 |
| Test | 2022-2025 |

Model tuning, teacher selection, and hyperparameter choices are made using the training and validation periods only. The test period is reserved for final evaluation.

## Models

The pipeline evaluates the following model classes:

- Logistic Regression baseline
- Random Forest baseline
- Extra Trees baseline
- LightGBM baseline
- AI teacher candidates: LightGBM and neural MLP
- AI-guided single-index VC student
- Unguided VC student ablation

## Quick start

### Option 1: Run from a cloned GitHub repository

In PowerShell:

```powershell
git clone https://github.com/Mahamoussa321/beyond-black-box-aaer-screening.git
cd beyond-black-box-aaer-screening

python -m venv .venv
.\.venv\Scripts\Activate.ps1

python -m pip install --upgrade pip
python -m pip install -r requirements.txt

python .\scripts\00_run_publication_pipeline.py *> publication_pipeline_log.txt
```

### Option 2: Run inside an already downloaded folder

Open PowerShell inside the project folder, then run:

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1

python -m pip install --upgrade pip
python -m pip install -r requirements.txt

python .\scripts\00_run_publication_pipeline.py *> publication_pipeline_log.txt
```

### macOS/Linux

```bash
python -m venv .venv
source .venv/bin/activate

python -m pip install --upgrade pip
python -m pip install -r requirements.txt

python scripts/00_run_publication_pipeline.py 2>&1 | tee publication_pipeline_log.txt
```

## Reproducibility notes

- The temporal split is fixed before model fitting.
- Hyperparameters and model selection are based on validation-period performance.
- The temporal test period is used only for final reporting.
- The full pipeline log is saved as `publication_pipeline_log.txt`.
- Generated tables and figures are written to the `outputs/` directory.
